SAE1.0102




Objectif/Descriptif :


L'objectif du projet est de créer un programme de classification automatique de dépêches aussi fiable et rapide que possible. Une dépêche est un court texte correspondant à une information journalistique.



Intitulé : Implémentation d'un besoin client et comparaison d'approches algorithmiques


Thème : Développement logiciel, exploitation de base de données


Nous avons créé un algorithme de classification automatique des dépêches par catégories (culture, économie, politique, sports et environnement-sciences) en utilisant un lexique créé automatiquement pour chaque catégorie. Ce lexique contient un ensemble de mots marqueurs obtenus en calculant les mots les plus fréquents par catégorie. Notre algorithme attribue un score aux mots du lexique en fonction de leur degré d'appartenance à la catégorie. Ensuite, notre algorithme compare chaque mot d'une dépêche et additionne le score de chaque mot du lexique présent dans la dépêche, ce qui nous permet d'obtenir un score global par catégorie pour une dépêche. Le score le plus élevé est très probablement la véritable catégorie de cette dépêche.
J'ai donc aidé à écrire le code principal de l'algorithme de classification et j'ai également créé quelques fonctions secondaires telles que le calcul du score ou la méthode de tri.



Attentes :


Il fallait fournir un algorithme de classification optimisé et un bilan global du projet en anglais. Nous avons respecté les attentes, avec quelques oublis tels que la comparaison entre l'algorithme réalisé au début du projet et celui optimisé à la fin du projet.

Intitulé : Création d'une base de données


Thème : Création de base de données, gestion de base de données, interprétation des résultats


Pour ce projet, nous avons reçu une base de données Excel ainsi qu'un scénario (le naufrage du Titanic). En fonction de ce que nous avons appris en lisant l'histoire du Titanic et en reliant cela aux chiffres que nous avons obtenus (comme le nombre de personnes par classe sauvées, le genre le plus sauvé, les tranches d'âge), nous avons pu formuler une conclusion. J'ai réalisé une grande partie de la base de données ainsi que les calculs de données, les mises en forme et les recherches. J'ai dû créer une base SQL et y inscrire les informations. Ensuite, j'ai trié ces données en utilisant des requêtes SQL


Attentes :


Il fallait rendre différents rendus pour chaque étape. Ces rendus étaient sous forme de questions qui nous ont été fournies et auxquelles il fallait répondre en utilisant des requêtes SQL. SAE2.04




Objectif/Descriptif :


Le but de ce projet est de comprendre les données qui nous sont fournies afin de pouvoir nettoyer et trier ces données pour en exporter le contenu voulu. Ensuite, créer des graphiques pour expliquer ces données.



Intitulé : Exploitation d'une base de données - Interprétation des données


Thème : Comprendre, nettoyer, trier et exporter des données, interpréter ces données


Ce projet était en groupe. Nous avons reçu un sujet nous précisant quelles données nous devons exporter parmi une énorme base de données. Nous travaillons sur une base de données SQL que nous avons créée afin d'y introduire les données présentes dans un fichier Excel. Après avoir compris ce que nous cherchions, nous avons d'abord trié les données afin d'y voir plus clair. Ces données recensent les accidents en France en fonction du type de voitures, des lieux, du genre, du nombre de victimes ainsi que des conditions de l'accident. Ensuite, nous avons nettoyé les données pour ne récupérer que ce qui nous est nécessaire. Lorsque nous avions toutes les informations nécessaires, nous avons transformé ces données en fichier .csv pour les exporter dans le logiciel RStudio. Dans RStudio, nous avons créé des graphiques interprétant les informations en plus de les décrire. Enfin, nous avons tout retranscrit dans un fichier .sql et un autre fichier PDF.
Dans ce projet, j'ai partagé le travail avec mon binôme de sorte que celui qui a fini puisse passer à la suite et/ou aider son binôme. Pour ma part, j'ai réalisé un peu plus de la moitié des créations de bases de données, ainsi que plusieurs requêtes. Pour la partie graphique, j'ai créé plusieurs graphiques et leurs descriptions.


Attentes :


Il fallait rendre plusieurs fichiers : Un premier fichier .sql contenant nos requêtes SQL ainsi que notre base de données. Ensuite, un deuxième fichier PDF comportant tous les graphiques et leurs explications.

Cela signifie créer une application sur laquelle il est possible de créer différents types d'événements (comme une soirée, une réunion, un mariage).



Intitulé : Conception et développement d'une plateforme d'hébergement d'événements


Thème : Conception de maquettes de site, développement d'une application, gestion de la charge de travail, applications mathématiques

Ensuite, nous avons effectué une gestion de projet en calculant la charge de travail pour déterminer les délais du projet. Par la suite, nous avons régulièrement travaillé sur la création d'un diagramme de classes ainsi que des cas d'utilisation pour définir les fonctionnalités de notre application. Ensuite, nous avons procédé à la conception d'une maquette et d'une image publicitaire pour définir la ligne directrice artistique que nous allions suivre. Ensuite, nous avons fait de même pour la maquette afin de visualiser à quoi ressemblerait l'application. Enfin, la dernière partie consistait au développement de notre application en Java : création des classes, des fonctions et vérification de leur fonctionnement.

J'ai également contribué aux diagrammes de classes et réalisé un diagramme de séquence. Ensuite, j'ai assisté l'un de mes camarades pour la fonction d'optimisation. Pour la partie logigramme et gestion de projet, nous avons tous participé simultanément à sa création. Enfin, pour la partie maquette, j'ai réalisé quelques pages et conçu le design du logo. Quant à la partie Java, j'ai créé quelques classes.


Attentes :


Dans ce projet, il fallait rendre de nombreux travaux : Un ensemble de diagrammes UML comprenant les diagrammes de classes, de séquences et de cas d'utilisation. Un diagramme de Gantt pour la gestion du projet, ainsi qu'un logigramme ultérieurement. Le développement en Java, ainsi que les maquettes et les images publicitaires de l'application.
